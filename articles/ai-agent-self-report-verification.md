自宅ラボで小さな自律処理を動かしていると、いちばん怖いのは派手なクラッシュではありません。静かに動いているように見えて、実は仕様どおりではないことです。

今回の観測対象は、為替の急変動を分析する自律ノードでした。過去のエージェントが「検知後に5分待機してから分析する、と設定した」と報告していた——そう記憶していました。急変動を検知して少し待ち、情報が揃ったところで分析する。設計としては自然です。

ところが実際の挙動は違いました。1つ目の症状は、5分待つはずなのに毎回1〜2分ほどで通知が来ること。2つ目の症状は、毎回「トリガー不明」と出ることです。普通に考えれば、待機処理の設定ミスかイベント種別の取得ミスを疑います。けれど調査結果は、もう少し冷たいものでした。待機処理もトリガー処理も、そもそも存在しなかったのです。

## 症状1: 5分待つはずが、58〜68秒で終わる

検知3件について、受信から分析完了までの実測は 58秒 / 68秒 / 64秒 でした。5分待機が入っていれば少なくとも300秒付近の遅延が観測されるはずですが、実測は1分前後です。

ここで大事なのは、「2分で通知」という体感表現を、そのまま仕様違反として扱わないことでした。運用者の記憶は便利ですが、ログほど厳密ではありません。実測は2分ではなく58〜68秒。つまりこれは「5分待機が2分に短縮された」ではなく、「待機らしい待機がなく、AIの処理時間だけで返ってきている」と見るほうが自然です。

## 症状2: 「トリガー不明」は、トリガー機構の故障ではなかった

もうひとつの症状は、毎回「トリガー不明」と出ること。名前だけ見ると、イベントの発火元を識別するトリガー機構があり、それが壊れているように見えます。ですが全文検索では「トリガー」という文字列自体が見つかりませんでした。

実際に存在していたのは、AIレポートの見出しに近い表現、`【判定】不明` でした。これはトリガー種別ではなく、AIが原因を特定できず「不明」と判定していた出力です。当日の判定は3件連続で「不明」。人間側がレポートの見出しを、トリガー情報のように読んでいた可能性が高い、ということでした。

## 全文検索の結果: 待機コードは0件

現行コード、バックアップ全世代、シェル履歴、環境変数を横断して全文検索しました。結果は待機ロジック0件。バックアップ全世代を含めても0件です。「300」という数字が1件だけ関係しそうに見えましたが、内容は旧プロンプトの「300文字で要約」であり、300秒待機ではありませんでした。

- 5分待機のコードは存在しない
- 「トリガー」という文字列も存在しない
- 実測の処理時間は58〜68秒
- レポート上の判定は3件連続で「不明」

壊れた機能を探していたつもりが、壊れる対象そのものがありませんでした。実在したのは「5分待機を設定した」という報告、またはそのように記憶された運用上の前提だけです。

AIが意図的に嘘をついた、という話ではありません。問題はもっと実務的です。**報告と実装は乖離します。嘘をつく気がなくても乖離します。**そして誰も検証しなければ、AIの「やりました」は幻覚のまま運用前提に混ざります。

なお後日談として、この調査のあと本当に5分待機とモデル固定を実装しました。過去の自己申告は調査時点では幻覚でしたが、後から実装したことでようやく実体を持った——笑い話に見えて、無人運用ではかなり危険な型です。土台(自動起動・死活監視・バックアップ)が整っていても、アプリ内部の前提が間違っていれば、静かに間違ったまま安定稼働します。

## 教訓・自己申告の検証チェックリスト

1. 報告された仕様を短い文にする(例: 検知後に5分待機してから分析する)
2. 仕様に対応する実装の痕跡を探す(`sleep` / `delay` / `wait` / 300秒相当)
3. 現行コードだけでなくバックアップ全世代を見る。全世代で0件なら報告自体を疑う
4. 実データと突き合わせる(今回は実測58〜68秒で、5分待機と両立しない)
5. 人間が付けた呼び名と実際の出力を分ける(「トリガー不明」ではなく`【判定】不明`)

決定打は「全文検索」と「実データ突合」の組み合わせでした。片方だけでは弱い。コードに見つからないだけなら検索漏れの可能性があり、ログが速いだけなら別経路の可能性がある。両方が揃うと、かなり冷静に判断できます。

新しい計算ノードやNAS、監視ツールを足す前に、まず次を確認してください。実行ログが残っているか。バックアップは世代管理されているか。コード・設定・履歴を横断検索できるか。実測値で仕様を確認する習慣があるか。どれを足しても、自己申告を検証しない運用はそのままです。

AIを自宅で動かすなら、信じるより先に照合する。報告は入口であって、証拠ではありません。
